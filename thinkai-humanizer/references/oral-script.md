# 口播文案适配规则

口播稿不仅要“读起来自然”，还要让听众第一次听就能理解。

## 可说性检查

逐句检查：

- 是否能在自然呼吸范围内说完
- 是否存在多层嵌套或连续修饰
- 是否有只适合阅读的书面连接词
- 是否容易读错、断错或产生歧义
- 是否连续堆叠数字、术语和条件
- 是否必须回看文字才能理解

一句话信息过多时，按意义拆句，不按固定字数机械切分。

## 听觉优先

口播稿应满足：

- 重点尽量提前
- 代词指向清楚
- 因果关系可以直接听懂
- 必要条件紧跟对应结论
- 专业术语首次出现时有简短解释
- 连续数字之间有明确归属

不要依赖括号、脚注、复杂层级和视觉排版传递关键信息。

## 节奏与停顿

- 长短句混合，不要全是碎短句。
- 判断句可以单独成句。
- 复杂信息之间用自然停顿隔开。
- 相邻句避免反复使用同一开头。
- 减少连续“然后、但是、所以、其实”。
- 保留原作者自然存在的少量口头习惯。

示例：

```text
原文：
它最大的优势并不是功能数量多，而是在通勤、办公以及临时外出等多个场景下都可以提供更加高效便捷的操作体验。

改写：
它的优势不是功能多，是用起来省事。通勤、办公，或者临时出门，都不用重新折腾设置。
```

## 开头处理

优先删除无效铺垫：

- 随着时代发展
- 在如今这个……的时代
- 今天我们来聊一聊
- 相信很多人都遇到过
- 众所周知

根据原稿选择合适开头：

- 直接结论
- 具体问题
- 使用场景
- 有依据的反常识判断
- 对目标用户最有用的信息

不要强制套用统一“前三秒钩子”。

## 结尾处理

可以用以下方式自然结束：

- 明确判断
- 适用或不适用人群
- 具体建议
- 限制或风险提醒
- 一个有实际意义的问题
- 停在最后一个有效事实

不强制：

- 总结全文
- 强行升华
- 引导点赞关注
- “你会选 A 还是 B”
- 固定三选一 CTA

## 口头禅边界

以下词不是见到就删：

- 其实
- 说实话
- 我觉得
- 就是
- 然后

如果它们符合原作者习惯且有节奏作用，可以保留。连续重复、没有意义或由改写者强行添加时才删除。

## 朗读自检

输出前默读或模拟朗读一遍：

- 有没有一口气说不完的句子
- 停顿位置是否符合意思
- 听众第一次听能否理解
- 是否出现绕口或容易读错的组合
- 重点是否被铺垫埋住
- 语气是否像原作者本人
- 是否为了口语化而变得过碎、过吵或过度表演
