# 事实与内容保护规则

降 AI 是表达优化，不得以自然化为理由改变事实、观点或作者身份。

## 必须锁定的内容

- 人名、组织名、品牌名、产品名、型号
- 价格、数量、比例、日期、时间、尺寸和单位
- 功能描述、技术参数、使用条件和兼容要求
- 适用范围、例外情况、限制条件和风险提示
- 专业术语、法律措辞、引用内容和来源归属
- 用户明确要求保留的句子或关键词
- 原文核心观点、立场、结论和信息量
- 有明确素材支持的真实经历

可以调整表达和位置，但不能改变含义、确定程度或适用边界。

## 确定程度保护

必须保留事实边界：

| 原文边界 | 不得改成 |
|---|---|
| 可能、通常、有助于 | 一定、必然、完全能够 |
| 最长可达 | 能达到、始终达到 |
| 官方称、资料显示 | 我实测、事实证明 |
| 在特定条件下 | 所有情况下 |
| 部分用户 | 大家、所有人 |

不要为了让语气更肯定而扩大结论。

## 禁止虚构

没有用户素材或可靠原文支持时，不得新增：

- “我用了三个月”等个人经历
- 测试次数、使用时长和测评结论
- 朋友、客户、粉丝或群体反馈
- 销量、评分、排名和比较数据
- 专家观点、调查结果和来源
- 产品未声明的功能和效果
- 原作者没有表达过的立场

可以把虚构式体验改成条件场景，但不能暗示它真实发生过。

示例：

```text
不合格：我连续用了一个月，续航完全没有问题。
可接受：按原文给出的续航时间，正常通勤不需要每天充电。
```

第二句成立的前提是原文确实提供了对应续航数据。

## 个人经历处理

- 用户明确提供的真实经历：保留事实，可优化表达。
- 来源不明的“我实测”：删除经历声明，保留有依据的信息。
- 假设场景：使用“如果、比如、在……时”等非事实化表述。
- 主观判断：可以保留“我觉得、我更在意”，不要自动改成客观事实。

## 删除边界

可以优先删除：

- 不含新信息的套话
- 重复表达的结论
- 没有事实支持的夸张评价
- 与正文无关的升华
- 模糊的群体背书

不得删除：

- 理解结论所需的原因
- 功能、条件、限制和风险
- 用户收益和使用场景中的必要信息
- 影响购买或行动判断的信息
- 用户明确要求保留的段落

不确定是否可以删除时，保留并只调整表达。

## 回查方法

改写完成后，将原稿与改写稿逐项比对：

1. 专有名词是否一致。
2. 数字、单位、日期和价格是否一致。
3. 条件、限制和确定程度是否一致。
4. 核心观点和立场是否一致。
5. 是否新增原文无法支持的事实。
6. 是否因精简而丢失影响判断的信息。
