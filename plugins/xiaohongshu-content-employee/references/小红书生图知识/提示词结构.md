# 简洁提示词结构

Prompt 的目标是像阿道夫成品图那样，让模型完成视觉设计，而不是把内部规划
翻译成一张说明书。锁定产品、主题和少量文字后，把构图空间留给模型。

## 为什么简洁提示词更有效

- 每页只有一个场景主题，模型容易建立明确的视觉中心。
- 默认 2 组、最多 3 组短中文文案，文字能成为画面元素，而不是正文截图。
- 使用真实生活、产品摄影或杂志感场景，画面有空间、光线和物件关系。
- 只给简短字体气质和审美方向，不规定每个元素的坐标、比例和字重。
- 明确“其余构图和场景自由发挥”，让模型自行完成构图和视觉层级。
- 明确“避免纯色信息卡”，防止画面退化为参数清单或说明书。

## Internal Plan

`style_anchor`、`page_role`、`shot_type`、`subject_position`、
`subject_scale`、`background_scene` 和 `text_zone` 继续用于结构化交接和页面
差异控制。

结构化字段只用于内部规划，不得逐项复述进最终 Prompt。最终 Prompt 只把这些
字段归纳成一句自然的真实感场景或视觉方向，不出现字段名、坐标、百分比、色号
清单和布局指令。

## Final Prompt Contract

每条最终 Prompt 只保留：

1. 真实产品参考图和产品身份，非产品页可省略。
2. 当前页的一个主题。
3. 一个真实感场景或清晰视觉方向。
4. 默认 2 组、最多 3 组短中文文案。
5. 简短字体气质和审美方向。
6. `3:4` 小红书成品图。
7. “其余构图和场景自由发挥”。
8. “避免纯色信息卡”。

## Product Page Pattern

```text
参考图片主体为{品牌、产品名和型号}，保持参考图中的包装、Logo、颜色、比例和
主要识别特征，其余构图和场景自由发挥。为小红书制作一张 3:4 成品图，用
{真实感场景或视觉方向}表现{当前页主题}。画面中自然呈现必须出现的中文文字
“{短文案一}”“{短文案二}”，字体{简短字体气质}，采用{简短审美方向}，避免
纯色信息卡和参数清单。
```

每一个以产品为主体的页面仍传入对应 `reference_image_paths`，不只限于首图。
参考图负责产品保真，Prompt 不再复述参考图 ID、路径、`supported_views` 或
内部构图字段。

## Non-Product Page Pattern

```text
为小红书制作一张 3:4 成品图，用{真实感场景或视觉方向}表现{当前页主题}。
画面中自然呈现必须出现的中文文字“{短文案一}”“{短文案二}”，字体
{简短字体气质}，采用{简短审美方向}，避免纯色信息卡和参数清单，其余构图和
场景自由发挥，不伪造产品外观。
```

## Keep It Compact

不得要求每张图生成 4 到 5 行精确中文。不得写“第一关、第二关、第三关”或
“步骤一、步骤二”这类未经选题要求的结构。默认不写逐项摄影参数、复杂道具
清单、长负面词、色号、坐标、主体比例、本地合成或后期排版步骤。

标题、品牌名和页面短文案都直接由生图模型生成。HTML 不叠加任何标题或正文
图片文字，不使用代码补字或确定性排版。

全部 Prompt 必须在同一次 Compose 调用中输出，并连同 `style_anchor`、每页
`page_id`、参考图对应关系和信息任务完整展示给用户。批准哈希与当前 Prompt 包
一致后，才把全部 Prompt 一次并发发送给所选模型；批准后不得在 Produce Executor
中改写 Prompt，也不做生成后图片相似度检查。
