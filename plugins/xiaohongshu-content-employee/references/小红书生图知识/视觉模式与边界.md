# 视觉执行边界

- 每个安装环境首次使用时列出全部可用图片渠道和具体模型，由用户选择并保存默认项。
- Compose 完成后展示全部最终 Prompt、参考图对应关系和信息任务，并等待用户批准。
- 批准哈希与当前 Prompt 包一致后，全部最终 Prompt 一次并发发送给已保存的默认模型。
- Prompt 固定写明 `3:4` 小红书成品图，请求使用所选渠道的默认竖版尺寸。
- 需要保持产品外观时，每一个产品主体页都绑定清晰的真实产品参考图；不同页面
  通过场景、景别、主体位置和信息任务变化构图，不改变产品身份。
- 模型原图直接进入 HTML，不用代码加字、裁切、抠图、叠图或重新合成。
- 图片返回后只确认请求状态为 `complete` 且文件实际存在，不读取或核对返回图片的
  像素尺寸；模型原图直接进入 HTML。
- 每个请求独立记录状态；全部计划图片均为 `complete` 且文件实际存在后，才进入
  HTML 和运行日志生成。`failed` 或 `uncertain` 禁止交付，也禁止自动重发。
- 用户明确要求更换模型时才重新选择；普通任务不重复询问渠道，只对完整 Prompt
  包进行一次批准，不逐条确认。
- 默认渠道不支持任务要求的真实产品参考图时，必须在付费请求前阻断并重新选择
  可用渠道，不得伪装产品保真或回退到未批准的图片方案。
